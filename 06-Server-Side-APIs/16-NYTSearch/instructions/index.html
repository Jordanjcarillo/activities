NY Times Mini Project

In preparation for our start of project #1, we are going to build a mini-project with our team this morning

1.0 GOAL

Our end goal is to use the NYT API to get articles and display them in an elegant bootstrap-layout similar to this:
The working URL for the final app.

2.0 BUILD PLAN (20 mins)

To tackle this project there will likely be 3 key phrases

  • Do a cursory review of the overall app: what's happening?
  • What resources are needed (bootstrap? anything else?)
  • What is the API being used? Do you need to register? Examine the API (hint see section 2.1)
  • Plan the steps to build:
  • a) Wireframe out the logic - user will type search parameters, then ajax call, then display
  • b) Have some of your team start building the HTML, put in placeholder results.
  • c) Have other members of your team do the API-call tests
  • d) Build components, test, merge them, test/debug overall app.

2.1 API (10 mins)

Take some time to examine the API and talk about it. The overview page shows the articlesearch.json call that we will be using.

All the parameters are in this link: https://developer.nytimes.com/docs/articlesearch-product/1/routes/articlesearch.json/get


3.0 BUILD HTML UI (90 mins)

Build the initial UI, and remember to have input boxes to capture the fields needed in the demo.

Once built, wire up the capturing of inputs and display them in the console.log

Once this is ready, now try to build the Query string needed for the call. The team member who tested out the API parameters should be involved here.

Examine the API data and wire-up the result output to show the matching data.

Remember to use a LOT of console.log comments to understand what is happening at each step of AJAX call.
